# 架构文档

## 系统架构
本项目采用模块化设计，主要分为以下几个核心模块：

### 1. 文件遍历模块
- 功能：递归遍历目录结构，收集所有文件路径
- 依赖：标准库的fs和path模块
- 边界：与.gitignore解析模块交互

### 2. .gitignore解析模块
- 功能：解析.gitignore文件，生成过滤规则
- 依赖：gitignore库（或自定义实现）
- 边界：与文件遍历模块交互，提供过滤判断

### 3. 文件读取模块
- 功能：读取文件内容，处理文本编码
- 依赖：标准库的fs模块
- 边界：与文件遍历模块交互，提供文件内容

### 4. 内容合并模块
- 功能：将所有文件内容合并为单个txt文件，添加文件路径标识
- 依赖：标准库的fs模块
- 边界：与文件读取模块交互，生成输出文件

### 5. CLI交互模块
- 功能：处理命令行参数，提供用户交互
- 依赖：clap库（Rust命令行解析库）
- 边界：作为入口，协调其他模块

## 数据流
1. CLI模块接收用户输入，确定目标目录和输出路径
2. 文件遍历模块开始递归遍历目录
3. .gitignore解析模块加载并解析规则
4. 文件遍历模块根据过滤规则筛选文件
5. 文件读取模块读取符合条件的文件内容
6. 内容合并模块将所有内容合并为单个txt文件
7. 输出结果到指定路径

## 技术栈
- 语言：Rust
- 依赖：
  - clap：命令行参数解析
  - gitignore：.gitignore规则解析
  - anyhow：错误处理
  - walkdir：目录遍历
