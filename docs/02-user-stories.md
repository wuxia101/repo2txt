# 用户故事

## 用户故事1：作为开发者，我希望将整个git仓库转换为单个txt文件
- 作为：开发者
- 需求：能够将git仓库中的所有文件转换为一个txt文件，方便代码审查或文档生成
- 价值：提高代码审查效率，简化文档生成流程

## 用户故事2：作为开发者，我希望遵循.gitignore规则过滤文件
- 作为：开发者
- 需求：转换过程中自动忽略.gitignore中指定的文件和目录
- 价值：避免包含不必要的文件，保持输出文件的简洁性

## 用户故事3：作为开发者，我希望自定义输出文件路径
- 作为：开发者
- 需求：能够指定输出txt文件的路径和名称
- 价值：灵活适应不同的使用场景

## 用户故事4：作为开发者，我希望查看转换进度
- 作为：开发者
- 需求：在转换过程中能够看到当前处理的文件和进度
- 价值：了解转换状态，处理大仓库时避免等待焦虑
