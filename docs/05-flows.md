# 流程文档

## 主要工作流程

### 流程1：文件转换流程
```
用户输入 → CLI解析参数 → 加载.gitignore规则 → 遍历目录 → 过滤文件 → 读取内容 → 合并内容 → 输出文件
```

#### 详细步骤：
1. 用户运行命令：`repo2txt --source ./myproject --output output.txt`
2. CLI模块解析命令行参数，获取源目录和输出路径
3. 加载源目录下的.gitignore文件，解析规则
4. 递归遍历源目录，收集所有文件路径
5. 对每个文件，检查是否匹配.gitignore规则
6. 对于未被忽略的文件，读取其内容
7. 将文件路径和内容添加到输出缓冲区
8. 所有文件处理完成后，将缓冲区内容写入输出文件
9. 显示转换结果，统计处理的文件数量

### 流程2：.gitignore规则匹配流程
```
文件路径 → 检查否定规则 → 检查普通规则 → 确定是否忽略
```

#### 详细步骤：
1. 获取文件的相对路径
2. 从后往前检查.gitignore规则
3. 如果匹配到否定规则（!开头），则不忽略
4. 如果匹配到普通规则，则忽略
5. 如果没有匹配到任何规则，则不忽略

### 流程3：错误处理流程
```
错误发生 → 记录错误信息 → 跳过当前文件 → 继续处理 → 最终报告
```

#### 详细步骤：
1. 处理文件时发生错误（如无法读取）
2. 记录错误信息，包括文件路径和错误原因
3. 跳过当前文件，继续处理下一个文件
4. 所有处理完成后，显示错误报告

## 异常处理
1. 源目录不存在：显示错误并退出
2. .gitignore文件不存在：使用默认规则（不忽略任何文件）
3. 输出文件无法写入：显示错误并退出
4. 二进制文件：默认跳过，可通过参数配置包含
